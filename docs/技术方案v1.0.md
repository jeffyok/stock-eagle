# 量化智能选股选基系统 — 技术方案 v1.0

> 项目代号：**Claw**（鹰爪——精准出击）  
> 创建时间：2026-05-03  
> 作者：剑飞 & 小飞

---

## 一、项目定位

个人量化决策辅助系统，核心价值：

1. **提高选股选基胜率** — 多策略+AI 交叉验证，减少主观判断失误
2. **实时跟踪风口** — 板块异动/资金异动/热搜异动，第一时间感知
3. **纪律化操作** — 信号触发+止盈止损，克服贪婪恐惧
4. **一站式管理** — 选股→跟踪→买入→持仓→卖出→复盘，闭环

## 二、数据源方案（免费 + 实时）

### 主数据源矩阵

| 数据类型 | 数据源 | 实时性 | 说明 |
|---------|--------|--------|------|
| **A股实时行情** | AKShare（东方财富源） | 秒级延迟 | 免费，覆盖全市场，实时行情接口稳定 |
| **A股实时行情（备用）** | Ashare（新浪/腾讯双源） | 秒级 | 自动故障切换，兜底方案 |
| **历史K线/日线** | AKShare + BaoStock | 日级 | BaoStock 历史数据最完整，免费无限制 |
| **财务报表** | AKShare（东方财富源） | 季报更新 | 免费完整 |
| **资金流向** | AKShare | 盘中实时 | 主力/散户/超大单分级别 |
| **北向资金** | AKShare | 日级 | 沪股通/深股通逐日数据 |
| **龙虎榜** | AKShare | 日级 | 营业部买卖明细 |
| **板块/概念** | AKShare | 实时 | 行业板块+概念板块涨跌幅 |
| **基金净值/估值** | AKShare + efinance | 实时估值 | 场外基金实时估值+历史净值 |
| **ETF行情** | AKShare | 实时 | 场内ETF行情+持仓 |
| **热搜/关注** | westock-data | 盘中 | 腾讯自选股热搜榜 |
| **宏观数据** | AKShare | 月/季级 | CPI/PPI/PMI/利率等 |

### 数据源架构

```
┌──────────────┐
│   Web UI      │  ← Streamlit 展示层
└──────┬───────┘
       │
┌──────▼───────┐
│  业务服务层   │  ← 策略引擎 / 智能引擎 / 风口监控
└──────┬───────┘
       │
┌──────▼───────┐
│  数据服务层   │  ← 统一数据接口（屏蔽底层数据源差异）
└──────┬───────┘
       │
┌──────┴───────┐
│  AKShare(主) │  Ashare(备)  BaoStock(历史)  efinance(基金)
└──────────────┘
```

**核心原则：所有数据通过统一数据服务层获取，上层业务不直接调用数据源，方便切换和维护。**

## 三、技术架构

### 技术栈

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| **语言** | Python 3.14 | 剑飞主力语言 |
| **Web框架** | Streamlit | 快速原型，交互丰富，适合数据展示 |
| **后端API** | FastAPI | 异步高性能，供前端调用+定时任务 |
| **数据库** | MySQL 8.x | 剑飞指定，支持复杂查询 |
| **ORM** | SQLAlchemy 2.x | Python 标准 ORM，异步支持 |
| **数据源** | AKShare + Ashare + BaoStock + efinance | 免费+实时 |

| **技术指标** | ta-lib / pandas-ta | 技术分析计算 |
| **回测引擎** | 自研轻量回测 | 满足基本回测需求 |
| **推送** | 飞书机器人 / 企业微信 | 信号通知 |

## 四、数据库设计

### 核心表结构

```sql
-- 股票基本信息
CREATE TABLE stock_basic (
    code        VARCHAR(10) PRIMARY KEY COMMENT '股票代码 sh600519',
    name        VARCHAR(20) NOT NULL COMMENT '股票名称',
    industry    VARCHAR(20) COMMENT '所属行业',
    market      ENUM('sh','sz','bj') COMMENT '市场',
    list_date   DATE COMMENT '上市日期',
    status      TINYINT DEFAULT 1 COMMENT '1-正常 0-退市',
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 日线行情
CREATE TABLE stock_daily (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    code        VARCHAR(10) NOT NULL,
    trade_date  DATE NOT NULL,
    open        DECIMAL(10,2),
    close       DECIMAL(10,2),
    high        DECIMAL(10,2),
    low         DECIMAL(10,2),
    volume      BIGINT COMMENT '成交量(手)',
    amount      DECIMAL(18,2) COMMENT '成交额',
    turnover    DECIMAL(10,4) COMMENT '换手率%',
    pct_chg     DECIMAL(10,4) COMMENT '涨跌幅%',
    UNIQUE KEY uk_code_date (code, trade_date),
    INDEX idx_trade_date (trade_date)
);

-- 实时行情快照
CREATE TABLE stock_realtime (
    code        VARCHAR(10) PRIMARY KEY,
    name        VARCHAR(20),
    price       DECIMAL(10,2) COMMENT '当前价',
    pct_chg     DECIMAL(10,4) COMMENT '涨跌幅%',
    volume      BIGINT,
    amount      DECIMAL(18,2),
    buy1_price  DECIMAL(10,2),
    sell1_price DECIMAL(10,2),
    high        DECIMAL(10,2),
    low         DECIMAL(10,2),
    open        DECIMAL(10,2),
    yesterday   DECIMAL(10,2) COMMENT '昨收',
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 资金流向
CREATE TABLE stock_money_flow (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    code        VARCHAR(10) NOT NULL,
    trade_date  DATE NOT NULL,
    net_mf      DECIMAL(18,2) COMMENT '主力净流入',
    net_mf_big  DECIMAL(18,2) COMMENT '大单净流入',
    net_mf_small DECIMAL(18,2) COMMENT '小单净流入',
    UNIQUE KEY uk_code_date (code, trade_date)
);

-- 龙虎榜
CREATE TABLE dragon_tiger (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    code        VARCHAR(10) NOT NULL,
    trade_date  DATE NOT NULL,
    reason      VARCHAR(100) COMMENT '上榜原因',
    buy_amount  DECIMAL(18,2) COMMENT '买入额',
    sell_amount DECIMAL(18,2) COMMENT '卖出额',
    net_amount  DECIMAL(18,2) COMMENT '净额',
    buy_depts   TEXT COMMENT '买入营业部(JSON)',
    sell_depts  TEXT COMMENT '卖出营业部(JSON)',
    INDEX idx_date (trade_date)
);

-- 板块行情
CREATE TABLE sector_daily (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    sector_code VARCHAR(10) NOT NULL,
    sector_name VARCHAR(20),
    sector_type ENUM('industry','concept') COMMENT '行业/概念',
    trade_date  DATE NOT NULL,
    close       DECIMAL(10,2),
    pct_chg     DECIMAL(10,4),
    net_mf      DECIMAL(18,2) COMMENT '板块资金净流入',
    lead_stock  VARCHAR(10) COMMENT '领涨股代码',
    UNIQUE KEY uk_sector_date (sector_code, trade_date)
);

-- 基金基本信息
CREATE TABLE fund_basic (
    code        VARCHAR(10) PRIMARY KEY COMMENT '基金代码',
    name        VARCHAR(50) NOT NULL,
    fund_type   VARCHAR(20) COMMENT '股票型/混合型/债券型/指数型',
    manager     VARCHAR(30) COMMENT '基金经理',
    size        DECIMAL(18,2) COMMENT '规模(亿)',
    establish_date DATE,
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 基金净值
CREATE TABLE fund_nav (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    code        VARCHAR(10) NOT NULL,
    nav_date    DATE NOT NULL,
    nav         DECIMAL(10,4) COMMENT '单位净值',
    acc_nav     DECIMAL(10,4) COMMENT '累计净值',
    pct_chg     DECIMAL(10,4) COMMENT '日涨跌幅%',
    UNIQUE KEY uk_code_date (code, nav_date)
);

-- 策略信号
CREATE TABLE strategy_signal (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    code        VARCHAR(10) NOT NULL,
    signal_type VARCHAR(20) NOT NULL COMMENT '策略类型: multi_factor/macd_golden/breakout...',
    direction   ENUM('buy','sell') NOT NULL,
    price       DECIMAL(10,2) COMMENT '信号价格',
    score       DECIMAL(5,2) COMMENT '信号强度0-100',
    reason      TEXT COMMENT '信号原因',
    signal_date DATETIME NOT NULL,
    is_expired  TINYINT DEFAULT 0,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_code_type (code, signal_type),
    INDEX idx_date (signal_date)
);

-- 持仓管理
CREATE TABLE portfolio (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    code        VARCHAR(10) NOT NULL,
    name        VARCHAR(20),
    asset_type  ENUM('stock','fund') DEFAULT 'stock',
    buy_price   DECIMAL(10,2) NOT NULL,
    buy_date    DATE NOT NULL,
    quantity    INT NOT NULL COMMENT '持有数量(股/份)',
    cost        DECIMAL(18,2) COMMENT '总成本',
    target_price DECIMAL(10,2) COMMENT '目标价',
    stop_price  DECIMAL(10,2) COMMENT '止损价',
    status      ENUM('holding','sold') DEFAULT 'holding',
    sell_price  DECIMAL(10,2),
    sell_date   DATE,
    profit      DECIMAL(18,2) COMMENT '盈亏金额',
    profit_pct  DECIMAL(10,4) COMMENT '盈亏比例%',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 每日复盘
CREATE TABLE daily_review (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    review_date DATE NOT NULL UNIQUE,
    market_trend TEXT COMMENT '大盘走势描述',
    hot_sectors TEXT COMMENT '热门板块(JSON)',
    signals     TEXT COMMENT '当日信号(JSON)',
    operation   TEXT COMMENT '操作建议',
    summary     TEXT COMMENT '复盘总结',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);
```
